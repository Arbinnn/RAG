# RAG (Retrieval-Augmented Generation) System

## What This Repo Does

This is a complete Retrieval-Augmented Generation (RAG) system that combines document management, semantic search, and AI-powered question answering. Users can upload documents in multiple formats, the system chunks and embeds them into a vector database, and then retrieves relevant content to answer natural language queries using a large language model.

### Key Features

- Multi-format document support: PDF, CSV, Excel (.xlsx, .xls), and Word (.docx)
- Automatic text chunking with configurable overlap
- Vector embeddings using OpenAI's text-embedding-3-large model
- Fast semantic search using Qdrant vector database
- AI-powered answers generated by GPT-4o-mini on GitHub Models
- Web-based UI built with Streamlit for easy document and query management
- Asynchronous workflow orchestration using Inngest
- Concurrent access support through remote vector database
- Document lifecycle management (upload, delete, clear database)

## How to Run It

### Prerequisites

- Python 3.12+
- Windows, macOS, or Linux
- Git

### Setup Steps

1. **Clone the repository**

   ```powershell
   git clone https://github.com/Arbinnn/RAG.git
   cd RAG
   ```

2. **Create and activate virtual environment**

   ```powershell
   python -m venv .venv
   .\.venv\Scripts\Activate.ps1
   ```

3. **Install dependencies**

   ```powershell
   pip install -r requirements.txt
   ```

   Or using poetry:

   ```powershell
   poetry install
   ```

4. **Configure environment variables**

   Create a `.env` file in the project root with:

   ```
   API_KEY=your_github_token_here
   OPENAI_BASE_URL=https://models.inference.ai.azure.com/
   QDRANT_URL=http://localhost:6333
   ```

   - `API_KEY`: GitHub Personal Access Token (generate at https://github.com/settings/tokens)
   - `OPENAI_BASE_URL`: Azure endpoint for GitHub Models (can use GitHub token)
   - `QDRANT_URL`: Optional remote Qdrant instance; if omitted, uses local storage

5. **Start the vector database**

   Using Docker (recommended for concurrent access):

   ```powershell
   docker run -p 6333:6333 qdrant/qdrant:latest
   ```

   Or set `QDRANT_URL` for an existing remote instance.

6. **Start Inngest dev server** (in a new terminal)

   ```powershell
   npx inngest-cli@latest dev -u http://127.0.0.1:8000/api/inngest --no-discovery
   ```

7. **Start FastAPI server** (in a new terminal)

   ```powershell
   python -m uvicorn main:app --host 127.0.0.1 --port 8000
   ```

8. **Start Streamlit frontend** (in a new terminal)
   ```powershell
   streamlit run streamlit_app.py
   ```

The Streamlit UI will open automatically at `http://localhost:8501`

## Architecture

### Components

- **FastAPI Backend** (port 8000): REST API with Inngest workflow endpoints
- **Inngest Dev Server** (port 8288): Serverless workflow orchestration and execution
- **Qdrant Vector Database** (port 6333): Vector similarity search
- **Streamlit Frontend** (port 8501): User interface for document management and queries

### Workflow

The system has two main workflows:

1. **Document Ingestion** (`rag/ingest_pdf`)
   - Load and chunk documents using LlamaIndex
   - Generate embeddings for each chunk using OpenAI API
   - Store embeddings in Qdrant with source metadata
   - Tracks chunks and source IDs for retrieval

2. **Query Processing** (`rag/query_pdf_ai`)
   - Embed the user question
   - Search vector database for relevant chunks
   - Generate answer using GPT-4o-mini with retrieved context
   - Return answer with source references

## Scope

### Supported Operations

- Upload documents in PDF, CSV, Excel, and Word formats
- View all uploaded documents in the library
- Delete individual documents or clear entire database
- Query across all documents with natural language questions
- Retrieve answer with referenced source documents
- Adjust number of context chunks returned per query

### Integrated Models

- **Embeddings**: OpenAI text-embedding-3-large (3072-dimensional vectors)
- **LLM**: GPT-4o-mini via GitHub Models API
- **Authentication**: GitHub Personal Access Token (auto-detected for routing)

## Limitations

### Known Constraints

- **Chunk Size**: Fixed at 1000 characters with 200 character overlap (configured in dataloader.py)
- **Concurrent Access**: Local Qdrant storage supports only single-process access; use remote Qdrant for concurrent access (can set QDRANT_URL)
- **Query Timeout**: 120 seconds per query before UI times out
- **File Upload Size**: Limited by Streamlit (default 200MB), can be configured in streamlit config
- **Vector Database**: Requires active Qdrant instance (local or remote)
- **API Rate Limits**: Subject to GitHub Models and OpenAI API rate limits
- **No Authentication**: Streamlit UI has no user authentication or access control
- **In-Memory Polling**: Inngest dev server stores workflows in memory; restarting clears execution history

### Document Format Limitations

- **PDF**: Extracts all text; image-based PDFs not supported
- **CSV/Excel**: Treats all values as text; no special handling for numbers or dates
- **Word**: Extracts text and tables; complex formatting lost
- **File Size**: Large files (>100MB) may cause performance issues during processing

### Content Retrieval Limitations

- Returns top K most similar chunks; may miss relevant context if chunks don't match query semantically
- No cross-document reasoning or multi-hop queries
- Vector search based on similarity; keyword search not available
- No real-time indexing; documents must be fully processed before querying

### Deployment Limitations

- **Development Only**: Inngest dev server is for local development; not suitable for production
- **No SSL**: HTTP-only communication; use reverse proxy for HTTPS in production
- **Data Persistence**: Qdrant can be local (single-machine only) or remote; FastAPI state not replicated
- **No Logging**: Limited audit trail; no user activity tracking

## Troubleshooting

### Port Already in Use

If a port is already bound (e.g., 8000, 8288, 6333), kill the process:

```powershell
Get-Process | Where-Object {$_.ProcessName -like "*python*" -or $_.ProcessName -like "*node*"} | Stop-Process -Force
```

### Qdrant Storage Already Accessed Error

This occurs when both local Qdrant and FastAPI try to access the same database file. Set `QDRANT_URL` to use remote Qdrant instead.

### "Push protection" error on git push

If you accidentally commit `.env`, use GitHub's secret scanning unblock feature or regenerate your token:

```powershell
git rm --cached .env
git add .gitignore
git commit -m "Remove .env from version control"
```

### Inngest Dev Server Won't Start

Ensure the path to FastAPI is correct:

```powershell
npx inngest-cli@latest dev -u http://127.0.0.1:8000/api/inngest --no-discovery
```
